    In case the failover failed we can restore the NFS exports in two ways:

    1. Add NFS exports one by one:

    add a new export⚓︎

    • mmnfs export add /gpfs/mariana/ --client "10.1.1.0/24(Access_Type=RW)"

    add access for another IP to the same nfs export⚓︎

    mmnfs export change /gpfs/mariana/ --nfsadd "10.23.23.23(Access_Type=RW)"

    remove access for an ip⚓︎

    mmnfs export change /gpfs/mariana/ --nfsremove "10.23.23.21"

    remove export⚓︎

    mmnfs export remove /gpfs/mariana/

    ## copy the existing conf >>> /var/mmfs/ces/nfs-config/gpfs.ganesha.exports.conf

    mmnfs export load /location/to/gpfs.ganesha.exports.conf ##this will start the NFS in this node and any other node that has nfs

    keytab creation⚓︎

    • From the command line, type ktutil to launch the utility.

    • Type the following command:

      addent -password -p -k 1 -e

      where is the Kerberos principal user. For example:

      addent -password -p la_wa -k 1 -e aes256-cts-hmac-sha1-96

    • When prompted, enter the password for the Kerberos principal user.

    • Type the following command to create a keytab:

      wkt /root/la_wa.keytab

    • Type q to quit the ktutil utility.

    • finally you can authenticate

      kinit -V -k -t /root/la_wa.keytab la_wa@INTRA.TTU.EE or kinit -V -k -t /root/ldaphpcbi.keytab ldaphpcbind@INTRA.TTU.EE

    Quotas⚓︎

    Query⚓︎

    # Quota Query

    mmlsfileset mariana -L mmlsquota -j smbgroup mariana --block-size G
    mmlsquota -u ahnasr mariana:home
    mmlsquota -d #defaul quota
    mmlsquota -u germo.vali --block-size auto | grep -v "no limits" mmrepquota -u mariana mmrepquota -j mariana mmrepquota -u -v mariana:home | column -t # users in home fileset mmcheckquota -v mariana

    mmedquota > manually changes the current quota

    mmdefedquota > changing the default quota for current users and new users whom are treated by the system as default users

    1. View Current Default Quotas > mmlsquota -d | column -t

    2. View current quota > mmrepquota mariana:home --block-size T| grep home | sort -k4n | column -t

    3. Default Quota Management > mmdefedquota -u mariana:home this define the default for newly created user or default users under home fileset (users with manually edited quotas will not be affected)

    4. Setting a fileset to the default quotas > mmedquota -d -j mariana:home (DO NOT USE SINCE ALL MANUAL QUOTA CHANGES WILL BE REVERTED)

    5. Changing inode max limit for a fileset > mmchfileset mariana smbgroup --inode-limit 10035200

    Fileset Management⚓︎

    • create a new fileset mmcrfileset keep in mind to specify the inode space and maxlimit otherwise it will be under root.

      mmcrfileset mariana cloud-backups --inode-space new -t "Cloud VMs backups fileset"

    • mmlinkfileset after creating a new fileset it has to be linked to visible for others.

      mmlinkfileset mariana cloud-backups -J /gpfs/mariana/nfs/cloud-backups

    • mmlsfileset mariana list all filesets and links status.

    • mmlsfileset mariana -L lists the inode limits.

    • mmdf mariana lists total inodes and storage sizes

    • mmrepquota -j mariana sizes of all the filesets.

    • mmchfileset mariana smbgroup --inode-limit 10035200 change the inode max limit for a fileset.
